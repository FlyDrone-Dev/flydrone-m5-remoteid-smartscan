# FlyDrone M5StickC Plus2 Remote ID Receiver — Smart Scan Edition

DJI等のドローンが発信するリモートID（ASTM F3411-19 / Wi-Fi Beacon）を受信し、
M5StickC Plus2 の画面に表示しつつ、BLE経由でスマホアプリ（BV Remote ID）へ
転送する受信機ファームウェアです。

本リポジトリは [`flydrone-m5-remoteid`](../flydrone-m5-remoteid) をベースに、
**Wi-Fiチャンネルのスキャン方式を「スマートスキャン方式」に変更した派生バージョン**です。

---

## 通常版との違い

| 項目 | 通常版 (flydrone-m5-remoteid) | スマートスキャン版 (本リポジトリ) |
|------|------|------|
| スキャン方式 | 1〜14chを常に等速でスイープ | 受信CHを記憶し、そのCHを集中スキャン |
| データ更新頻度 | 約7秒に1回（14ch均等のため低密度） | 受信CHに集中するため高密度・高頻度 |
| 新規CH探索 | 常時 | 定期的に全chを3周して探索 |
| BLE送信 | あり | あり（通常版と同一・変更なし） |
| LCD表示 | 複数機リスト表示 | 複数機リスト表示（通常版と同一） |

スマホへ送るデータ・画面表示は通常版と同じです。**変わるのはチャンネルの回し方だけ**です。

---

## スマートスキャンの動作

ドローンのリモートIDは特定のWi-Fiチャンネルでのみ発信されます。
全チャンネルを均等にスイープすると、目的のチャンネルに戻るまで時間がかかり、
データ更新が遅くなります。本方式は「当たりのチャンネル」を記憶して集中的に
回すことで、スマホへ送るデータの密度と更新頻度を上げます。

### 2つのフェーズ

```
┌─────────────────────────────────────────────┐
│  PHASE_SEARCH（探索フェーズ）                 │
│  1〜14ch を3周スイープ                         │
│       │                                       │
│       ├─ 受信なし → さらに3周（受信するまで） │
│       └─ 受信あり → 受信CHを記憶 → FOCUSへ    │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│  PHASE_FOCUS（濃密フェーズ）                  │
│  記憶したCHのみを30周スイープ（高密度受信）   │
│       │                                       │
│       ▼                                       │
│  1〜14ch を3周スイープして新規CHを探索        │
│  （新たに見つかったCHは記憶に追加）           │
│       │                                       │
│       └─ FOCUSに戻る（記憶CHを30周）          │
└─────────────────────────────────────────────┘
```

### 用語の定義

- **1スイープ** = 1ch〜14chを1周（14回のチャンネル切替）
- **探索フェーズの3周** = 全チャンネルを3回スイープ
- **濃密フェーズの30周** = 記憶したCHのみを30回スイープ

### 動作の流れ（例）

1. 起動直後は探索フェーズ。1〜14chを3周スイープ
2. 6chでリモートIDを受信 → `6ch` を記憶
3. 濃密フェーズへ移行。6chのみを30周スイープ（高速にデータ取得）
4. 30周完了後、1〜14chを3周スイープして新規CHを探索
5. 11chでも別ドローンを受信 → `11ch` を記憶に追加
6. 濃密フェーズに戻り、6chと11chを30周スイープ
7. 以降、4〜6を繰り返す

## ビルド・書き込み

### 必要な環境

- VS Code + PlatformIO IDE
- M5StickC Plus2 とデータ転送対応のUSB-Cケーブル

### 手順

1. VS Codeで本フォルダを開く
2. 下部ステータスバーの「✔（ビルド）」でコンパイル
3. M5StickC Plus2をUSB接続
4. 「→（Upload）」で書き込み
5. 「🔌（シリアルモニター）」で動作ログを確認

### platformio.ini

```ini
[env:m5stick-c-plus2]
platform = espressif32
board = m5stick-c-plus2
framework = arduino
upload_speed = 1500000
monitor_speed = 115200

lib_deps =
    m5stack/M5StickCPlus2 @ ^1.0.0
```

## 操作方法

| ボタン | 機能 |
|--------|------|
| Aボタン | 表示モード切替（LIST / DETAIL / STATS） |
| Bボタン | 受信データのリセット |

### 画面表示

- **LIST**：電波の強い順に最大3機を一覧表示（JU登録記号・シリアル番号・RSSI・CH）
- **DETAIL**：最後に受信した機体の詳細
- **STATS**：累計受信数・アクティブ機体数・スキャン状態

## 主要な設定（config.h）

| 定数 | 既定値 | 説明 |
|------|--------|------|
| `WIFI_CHANNEL_MIN` | 1 | スキャン最小チャンネル |
| `WIFI_CHANNEL_MAX` | 14 | スキャン最大チャンネル |
| `MAX_DRONES` | 20 | 同時追跡可能な最大機体数 |
| `DRONE_TIMEOUT_MS` | 10000 | 電波断後に画面から消えるまでの時間（ms） |
| `DISPLAY_UPDATE_INTERVAL` | 500 | 画面更新間隔（ms） |

スマートスキャン関連の周回数（探索3周・濃密30周）は `main.cpp` 内で定義しています。
チューニングする場合は該当の定数を調整してください。

## リモートIDデータ構造（参考）

受信payloadの先頭1バイトはメッセージカウンターで、実際のOpenDroneID
メッセージは2バイト目（Message Pack ヘッダ `0xF0`）から始まります。

```
payload[0]    : メッセージカウンター（0x70, 0x71... 解析対象外）
payload[1]    : Message Pack ヘッダ（0xF0）
payload[2]    : ブロックサイズ（25）
payload[3]    : ブロック数
payload[4-]   : Basic ID #1（シリアル番号 / ID Type=1）
                Basic ID #2（JU登録記号 / ID Type=2）
                Location, Authentication ...
```

ID Type の定義：

| Type | 内容 |
|------|------|
| 1 | Serial Number（製造シリアル番号） |
| 2 | CAA Assigned Registration ID（JU登録記号） |
| 3 | UTM Assigned UUID |
| 4 | Specific Session ID |
